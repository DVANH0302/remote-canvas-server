What's happening:

Opening a FIFO for writing with O_WRONLY (no O_NONBLOCK) blocks until a reader opens the other end. The client hasn't opened FIFO_S2C_<pid> for reading yet — it's waiting for SIGUSR2, which the server already sent from the signal handler. But the client also won't open the FIFOs until after receiving SIGUSR2. So:

Server's main loop is blocked on open(s2c, O_WRONLY)
Client received SIGUSR2 and tries to open FIFO_C2S_<pid> for writing — also blocks because the server's fd_read = open(c2s, O_RDONLY | O_NONBLOCK) already completed (non-blocking read side opened fine)
Client then tries to open FIFO_S2C_<pid> for reading — this would unblock the server... but the client is blocked on FIFO_C2S_<pid> write first
Actually the deadlock is: the server opens s2c for write (blocking, waiting for reader), while the client opens c2s for write (blocking, waiting for reader). Neither unblocks the other. Classic FIFO open deadlock.

The fix: open both FIFOs with O_NONBLOCK first, then use fcntl to clear the flag, or better — open both with O_RDWR or open in the right order. The simplest correct fix is to open both sides non-blocking:

c
// In main loop - open both non-blocking to avoid deadlock
int fd_read  = open(c2s, O_RDONLY | O_NONBLOCK);
int fd_write = open(s2c, O_WRONLY | O_NONBLOCK);  // add O_NONBLOCK here

if (fd_read == -1 || fd_write == -1) {
    // handle error
}

// Now clear O_NONBLOCK on fd_write if you want blocking writes later
int flags = fcntl(fd_write, F_GETFL);
fcntl(fd_write, F_SETFL, flags & ~O_NONBLOCK);

add_client(cpid, fd_read, fd_write);
Opening O_WRONLY | O_NONBLOCK on a FIFO with no reader yet will fail with ENXIO, so actually the cleanest approach is to also open fd_read first (which succeeds non-blocking), then the client opening the write end of c2s will unblock nothing since there's already a reader — but the server's open(s2c, O_WRONLY) still blocks until the client opens s2c for read.

The real fix is to open s2c as O_RDWR — opening a FIFO O_RDWR never blocks:

c
int fd_read  = open(c2s, O_RDONLY | O_NONBLOCK);
int fd_write = open(s2c, O_RDWR);   // O_RDWR on a FIFO never blocks
This is the minimal one-line change. O_RDWR on a named pipe is technically non-portable per POSIX but works on Linux, is commonly used for exactly this pattern, and avoids the blocking issue entirely. Your reads on fd_read already handle the O_NONBLOCK side properly.

Yes, pause() is definitely a problem. Here's what's happening:

The race condition with pause():

SIGUSR1 arrives → signal handler runs, sets new_client_pid, sends SIGUSR2
Signal handler returns → pause() is supposed to return
BUT — pause() only returns when a signal is received while it's waiting. If the signal arrives just before pause() is called, the signal is already handled and pause() blocks forever waiting for the next signal
This is a classic POSIX race. The fix is to use sigsuspend() with a proper signal mask instead of pause(). Let me look at the full main loop to give you the exact fix:

#define _POSIX_C_SOURCE 200809L

#include "client.h"
#include "queue.h"
#include "worker.h"
#include "scheduler.h"
#include <stdio.h>
#include <stdlib.h>
#include <signal.h>
#include <unistd.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <pthread.h>

struct RequestQueue queue;
pthread_mutex_t queue_mutex = PTHREAD_MUTEX_INITIALIZER;
pthread_cond_t not_empty = PTHREAD_COND_INITIALIZER;
pthread_cond_t not_full = PTHREAD_COND_INITIALIZER;

static volatile sig_atomic_t new_client_pid = 0;

// Block SIGUSR1 in main before calling sigsuspend,
// so the signal can't sneak in between the check and the wait
static sigset_t sigusr1_blocked;

void sigusr1_handler(int sig, siginfo_t *info, void *context) {
    (void)sig; (void)context;
    pid_t client_pid = info->si_pid;

    char c2s[32], s2c[32];
    snprintf(c2s, sizeof(c2s), "FIFO_C2S_%d", client_pid);
    snprintf(s2c, sizeof(s2c), "FIFO_S2C_%d", client_pid);

    unlink(c2s);
    unlink(s2c);
    mkfifo(c2s, 0666);
    mkfifo(s2c, 0666);

    if (kill(client_pid, SIGUSR2) == -1) {
        perror("kill failed");
        return;
    }
    new_client_pid = client_pid;
}

int main(int argc, char *argv[]) {
    if (argc != 2) {
        fprintf(stderr, "Not correct argument\n");
        return 1;
    }

    int num_threads = atoi(argv[1]);
    if (num_threads <= 0) {
        fprintf(stderr, "Invalid number of threads\n");
        return 1;
    }

    printf("Server PID: %d\n", getpid());
    fflush(stdout);

    // Block SIGUSR1 in the main thread's mask BEFORE installing the handler.
    // sigsuspend will atomically unblock it while sleeping.
    sigemptyset(&sigusr1_blocked);
    sigaddset(&sigusr1_blocked, SIGUSR1);
    pthread_sigmask(SIG_BLOCK, &sigusr1_blocked, NULL);

    struct sigaction sa;
    sa.sa_sigaction = sigusr1_handler;
    sa.sa_flags = SA_SIGINFO;
    sigemptyset(&sa.sa_mask);
    if (sigaction(SIGUSR1, &sa, NULL) == -1) {
        perror("sigaction");
        return 1;
    }

    initializeQueue(&queue);

    pthread_t *threads = malloc(num_threads * sizeof(pthread_t));
    for (int i = 0; i < num_threads; i++) {
        pthread_create(&threads[i], NULL, worker, NULL);
    }

    pthread_t scheduler;
    pthread_create(&scheduler, NULL, run_scheduler, NULL);

    // Build the mask for sigsuspend: everything EXCEPT SIGUSR1
    // so sigsuspend atomically unblocks SIGUSR1 and sleeps
    sigset_t wait_mask;
    sigfillset(&wait_mask);
    sigdelset(&wait_mask, SIGUSR1);

    while (1) {
        // Atomically: unblock SIGUSR1 and suspend.
        // No race window between checking new_client_pid and sleeping.
        sigsuspend(&wait_mask);

        while (new_client_pid != 0) {
            pid_t cpid = new_client_pid;
            new_client_pid = 0;

            char c2s[32], s2c[32];
            snprintf(c2s, sizeof(c2s), "FIFO_C2S_%d", cpid);
            snprintf(s2c, sizeof(s2c), "FIFO_S2C_%d", cpid);

            // O_RDONLY|O_NONBLOCK: returns immediately even with no writer yet
            int fd_read  = open(c2s, O_RDONLY | O_NONBLOCK);
            // O_RDWR on a FIFO never blocks regardless of whether
            // the other end is open yet
            int fd_write = open(s2c, O_RDWR);

            if (fd_read == -1 || fd_write == -1) {
                perror("open FIFO");
                if (fd_read != -1)  close(fd_read);
                if (fd_write != -1) close(fd_write);
                continue;
            }

            add_client(cpid, fd_read, fd_write);
        }
    }

    free(threads);
    return 0;
}
The two key changes:

1. sigsuspend instead of pause — pthread_sigmask(SIG_BLOCK, ...) blocks SIGUSR1 in the main thread before the loop. Then sigsuspend(&wait_mask) atomically unblocks SIGUSR1 and suspends in one indivisible operation. There is literally no window where the signal can arrive and be missed.

2. O_RDWR on s2c — Opening a FIFO with O_RDWR never blocks on Linux because the file descriptor counts as both a reader and a writer already present. O_WRONLY blocks until a reader opens the other end, which is what was causing the original timeout.

Also note: pthread_sigmask is used instead of sigprocmask because you have threads — sigprocmask behaviour is undefined in multithreaded programs. Using pthread_sigmask ensures only the main thread has SIGUSR1 unblocked via sigsuspend, while worker/scheduler threads inherit the blocked mask and won't accidentally catch the signal.
